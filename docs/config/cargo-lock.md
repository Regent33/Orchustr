# Cargo Lock

## File

- Path: `Cargo.lock`
- Purpose: lock exact resolved dependency versions for reproducible builds

## What It Contains

The lockfile records the exact versions selected for workspace dependencies such as `tokio`, `reqwest`, `serde`, `schemars`, `opentelemetry`, `sqlx`, `pyo3`, and `napi`. It is generated by Cargo rather than hand-authored.

## How to Read It

- Each package stanza names the crate, version, source, and dependency edges.
- Workspace crates appear alongside third-party crates.
- The file should change when dependency versions or feature resolution change.

⚠️ Known Gaps & Limitations
- This page does not inline the entire lockfile because it is generated data rather than a maintained design document.
- The repository does not currently pair the lockfile with a separate dependency update policy document.
